# /feature-dev 需求开发（轻量版）

4 阶段快速开发流程，适合不需要完整文档输出的快速迭代。

## 触发方式
```
/feature-dev <需求描述>
```

## 阶段 1：需求分析
- 理解需求，确认技术栈和目标平台
- 列出功能点和验收标准
- 门禁：用户确认

## 阶段 2：架构设计
- 委托 arch-designer 子代理
- 产出模块划分 + 关键类图/时序图
- 标注使用的架构模式和设计模式
- 门禁：用户选择方案

## 阶段 3：编码实现
- 最小改动原则实现
- 遵循项目编码规范
- 门禁：代码语法正确

## 阶段 4：代码审查
- 委托 code-reviewer 子代理
- 四维审查：正确性、性能、安全、可维护性
- 门禁：无高危问题
